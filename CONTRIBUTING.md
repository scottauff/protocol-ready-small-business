# CONTRIBUTING GUIDELINES

## GOAL
Improve the Protocol-Ready Small Business Standard through:
- UX testing
- operational validation
- transparency template improvements
- onboarding refinements
- schema updates
- real-world case studies

## CONTRIBUTION TYPES
- bug reports (UX friction)
- documentation improvements
- catalog schema refinements
- Lightning UX suggestions
- dispute model validation
- real customer onboarding examples
- SOPs for small business adoption
- localization or translations

## REWARD MODEL
All approved contributions may receive sats via:
- zap
- LN invoice
- V4V reward stream

## PROCESS
1. Open an issue
2. Submit PR or notes
3. Participate in review thread
4. Once merged:
   - contributor signs with Nostr key
   - TechHammer signs the merge
5. Reputation accrues to your key long term
